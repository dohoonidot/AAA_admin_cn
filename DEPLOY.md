# 서버 배포 시 체크리스트

## 1. 서버에 올리는 파일 (코드)

- **프로젝트 전체**를 그대로 배포하면 됩니다. (git clone / git pull 또는 압축 복사)
- 포트(3003/9998)는 **코드 수정 없이** 환경 변수로만 구분됩니다.

---

## 2. 서버에서만 설정/변경하는 것

### 2-1. `.env` 파일 (서버 전용으로 설정)

서버에는 **로컬과 다른 .env**를 둡니다. 아래만 서버에 맞게 넣으면 됩니다.

| 변수 | 로컬 (참고) | 서버 | 비고 |
|------|-------------|------|------|
| **PORT** | 3003 또는 미설정(NODE_ENV=development 시 3003) | **9998** 또는 미설정(기본 9998) | 서버는 9998 사용 |
| **NODE_ENV** | development (로컬 3003 쓰려면) | **설정 안 함** 또는 production | 서버에 development 넣지 않기 |
| **PGHOST** | localhost 등 | 서버 DB 호스트 | DB 주소 |
| **PGUSER** | - | DB 사용자 | |
| **PGPASSWORD** | - | DB 비밀번호 | |
| **PGDATABASE** | - | DB 이름 | |
| **PGPORT** | 5432 등 | DB 포트 | 필요 시만 |
| **JWT_SECRET** | - | 강한 비밀키 | 운영용으로 반드시 변경 |
| **UPSTREAM_HOST** | localhost | 실제 API 서버 주소 | 필요 시 |
| **UPSTREAM_PORT** | 8060 | 실제 API 서버 포트 | 필요 시 |
| **GIFT_UPSTREAM_HOST** | - | 선물 API 호스트 | 사용 시 |
| **GIFT_UPSTREAM_PORT** | - | 선물 API 포트 | 사용 시 |

**요약:** 서버에서는  
- `PORT=9998` 넣거나, 아예 안 넣어도 됨(기본 9998).  
- `NODE_ENV=development` 는 **넣지 않기**.

---

## 3. 서버에서 할 작업 순서

1. 프로젝트 올리기: `git clone` / `git pull` 또는 파일 복사
2. 의존성 설치: `npm install`
3. **서버용 .env** 만들기: 위 표 참고해서 작성 (PORT=9998 또는 생략, NODE_ENV 미설정 등)
4. 실행: `npm start` 또는 `node server/server.js`  
   (실서비스는 pm2 등으로 실행 권장)
5. 방화벽/보안그룹에서 **9998 포트** 열기 (필요 시)

---

## 4. 코드에서 바꿀 파일은?

- **없습니다.**  
  같은 코드로 로컬(3003) / 서버(9998) 구분은 **.env와 NODE_ENV** 로만 합니다.
